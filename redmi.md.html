Guida all'installazione
e configurazione

Dall'installazione di HACS alla prima vista Lovelace generata dall'AI. Tempo stimato: 10–15 minuti.

✓ Testato su HA 2024.1+

Prima di iniziare

Cosa ti serve

Requisito
Home Assistant installato e funzionante
Versione 2023.1 o superiore. HA OS, Container, Supervised o Core vanno tutti bene.
Requisito
Claude Desktop installato sul tuo PC
Scaricalo gratuitamente da claude.ai/download. Funziona su Windows e Mac.
Requisito
Account HA Intelligence con API Key
Registrati su ha-intelligence.netlify.app e copia la tua API Key dal pannello.
Requisito
Python 3.10+ sul PC
Il server MCP gira localmente in Python. Verifica con python --version nel terminale.

Step 01

Installa HACS in Home Assistant

HACS (Home Assistant Community Store) è il gestore di integrazioni della community. Se lo hai già, salta al passo 2.

1.1
Apri il terminale di Home Assistant
Vai su Impostazioni → Add-on → Terminal & SSH. Se non è installato, aggiungilo prima dall'Add-on Store.
1.2
Esegui lo script di installazione HACS
Copia e incolla nel terminale:
wget -O - https://get.hacs.xyz | bash -
1.3
Riavvia Home Assistant e abilita HACS
Dopo il riavvio vai su Impostazioni → Dispositivi e servizi → Aggiungi integrazione e cerca "HACS". Segui la procedura OAuth con GitHub.

Step 02

Installa l'integrazione HA Intelligence

2.1
Aggiungi il repository personalizzato in HACS
In HACS clicca i 3 puntini in alto a destra → Repository personalizzati. Inserisci:
https://github.com/TUO_USERNAME/ha-intelligence-hacs
Categoria: Integrazione
2.2
Installa l'integrazione
Cerca "HA Intelligence" in HACS → Installa → Riavvia HA.
2.3
Aggiungi l'integrazione in HA
Impostazioni → Dispositivi e servizi → Aggiungi integrazione → HA Intelligence. Inserisci la tua API Key quando richiesta.

Step 03

Copia la tua API Key

La trovi nel tuo pannello HA Intelligence, sezione API Key.

3.1
Accedi al pannello
Vai su ha-intelligence.netlify.app/dashboard.html → sezione API Key nella barra laterale.
3.2
Copia la chiave
Clicca Copia. La chiave ha il formato hai_live_xxxxxxxxxxxx. Salvala in un posto sicuro, ti serve nei prossimi step.
⚠ Attenzione: Non condividere la tua API Key. Ogni richiesta autenticata scala il tuo piano mensile.

Step 04

Crea un Long-Lived Access Token in HA

Questo token permette al server MCP di leggere le entità e scrivere la configurazione Lovelace.

4.1
Apri il profilo utente in Home Assistant
Clicca sull'icona del tuo profilo in basso a sinistra nella barra laterale di HA (o vai su homeassistant.local/profile).
4.2
Crea un nuovo token
Scorri in fondo alla pagina del profilo → Token di accesso a lungo termine → Crea token. Dagli un nome tipo HA Intelligence MCP.
4.3
Copia il token immediatamente
HA mostra il token solo una volta. Copialo subito e salvalo insieme all'API Key del punto precedente.
Importante: Il token dà accesso completo al tuo HA. Non includerlo in file pubblici o repository GitHub.

Step 05

Configura Claude Desktop

Il server MCP gira localmente sul tuo PC e si collega a Claude Desktop tramite il file di configurazione.

5.1
Scarica e posiziona il file mcp_server.py
Scarica mcp_server.py dalla sezione Release del repository GitHub. Mettilo in una cartella stabile, es.:
# Windows
C:\Users\TUO_NOME\ha-intelligence\mcp_server.py

# Mac / Linux
~/ha-intelligence/mcp_server.py
5.2
Installa le dipendenze Python
Apri il terminale nella cartella e installa i pacchetti necessari:
pip install mcp aiohttp pyyaml
5.3
Trova il file di configurazione di Claude Desktop
Il percorso è:
# Mac
~/Library/Application Support/Claude/claude_desktop_config.json

# Windows
%APPDATA%\Claude\claude_desktop_config.json
Se il file non esiste, crealo.
5.4
Aggiungi la configurazione MCP
Incolla questo contenuto nel file (sostituisci i valori con i tuoi):
{
  "mcpServers": {
    "ha-intelligence": {
      "command": "python",
      "args": ["/percorso/completo/mcp_server.py"],
      "env": {
        "HA_URL":     "http://homeassistant.local:8123",
        "HA_TOKEN":   "IL_TUO_HA_TOKEN_QUI",
        "HAI_API_KEY": "hai_live_LA_TUA_API_KEY_QUI"
      }
    }
  }
}
Percorso su Windows: usa doppi backslash: C:\\Users\\Nome\\ha-intelligence\\mcp_server.py
5.5
Riavvia Claude Desktop
Chiudi e riapri Claude Desktop. Nella barra di testo del chat dovresti vedere l'icona 🔨 (strumenti) — significa che l'MCP è attivo.

Step 06

Crea la tua prima dashboard

6.1
Chiedi una vista ad HA Intelligence
Apri Claude Desktop e scrivi in chat:
Crea una vista luci per il soggiorno, stile glass, con tutte le luci disponibili.
L'AI leggerà le entità dal tuo HA, progetterà la vista e la salverà in staging. Vedrai il YAML generato direttamente in chat.
6.2
Chiedi variazioni (opzionale)
Prima di pubblicare, puoi modificare: "Aggiungi anche la scena Cinema", "Usa icone più grandi", "Togli la luce del corridoio". L'AI aggiornerà lo staging.
6.3
Pubblica
Quando sei soddisfatto scrivi:
Pubblica
La vista appare in Home Assistant immediatamente. HA Intelligence fa un backup automatico prima di qualsiasi scrittura.
✓ Installazione completata! Ora puoi creare tutte le viste che vuoi. Il piano Free include 5 richieste al mese.

FAQ

Domande frequenti

L'AI può modificare automazioni o script esistenti?
No. HA Intelligence scrive esclusivamente viste Lovelace. Non tocca automazioni, script, configurazioni YAML di HA o altri file del sistema.
Cosa succede se publico per errore una vista sbagliata?
Prima di ogni pubblicazione viene salvato un backup automatico della configurazione Lovelace esistente. Puoi ripristinare chiedendo all'AI: "Ripristina il backup precedente della vista Soggiorno".
HA è in YAML mode, cosa cambia?
HA Intelligence funziona solo con Lovelace in modalità Storage (la predefinita). Se hai attivato la modalità YAML manuale, devi prima disattivarla da Impostazioni → Dashboard → ⋮ → Prendi il controllo.
HA_URL funziona anche fuori casa?
Se accedi a HA da remoto, usa il tuo URL Nabu Casa (https://xxx.ui.nabu.casa) o il tuo dominio con tunnel (Cloudflare, DuckDNS, ecc.). Il HA_TOKEN rimane lo stesso.
Claude Desktop non mostra l'icona strumenti
1) Controlla che il percorso in claude_desktop_config.json sia assoluto e corretto. 2) Verifica che Python sia nel PATH con python --version. 3) Testa il server manualmente: python /percorso/mcp_server.py non deve dare errori all'avvio. 4) Riavvia completamente Claude Desktop.
